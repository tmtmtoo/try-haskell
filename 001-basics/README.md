# 001 — 基礎文法

## この章で何ができるようになるか

- 簡単な関数を書ける／読める
- 「パターンマッチ」「ガード」「再帰」のような Haskell らしい書き方が一通り分かる
- リストを再帰や高階関数で操作できる
- 「`$` と `.` ってなに?」に答えられる

## まず一行のメンタルモデル

Haskell の関数定義は **数学の `f(x) = ...` をそのまま書いている**、と思って読み始めるといい。`if-else` を書く代わりに「パターンごとに `=` を並べる」のが基本スタイル。

---

## 1. 関数定義

```haskell
add :: Int -> Int -> Int
add x y = x + y
```

- 1 行目: **型シグネチャ**。「`Int` を 2 つもらって `Int` を返す」
- 2 行目: **定義**。引数を空白で並べる（`add(x, y)` ではなく `add x y`）

呼び出しも空白:

```haskell
add 3 4   -- → 7
```

### カリー化 — 部分適用ができる

`Int -> Int -> Int` は **`Int -> (Int -> Int)`** のこと。つまり「`Int` を受け取って『`Int` を受け取って `Int` を返す関数』を返す」関数。

```haskell
add3 :: Int -> Int
add3 = add 3        -- 引数を 1 つだけ与えた状態で「固定」する

add3 4   -- → 7
```

これを **部分適用** と呼ぶ。Go や Python の `partial` に相当するが、Haskell では「全部の関数が最初からこう」。

> **ポイント**: `f x y` を「2 引数の関数呼び出し」ではなく「`f x` してできた関数を `y` に適用する」と読み替えられるようになると、後の章が一気にラクになる。

---

## 2. パターンマッチとガード

### パターンマッチ

入力の形に合わせて `=` の右側を切り替える。

```haskell
describe :: Int -> String
describe 0 = "zero"
describe 1 = "one"
describe _ = "many"     -- _ は「何でもいい」を意味するワイルドカード
```

上から順に試して、最初に一致したものが採用される。**他の言語の `switch` に近いが、引数そのもののパターン** になる。

リストを分解するパターン:

```haskell
firstOf :: [Int] -> Int
firstOf []     = 0           -- 空リスト
firstOf (x:_)  = x           -- 先頭 x、残り任意
```

`(x:xs)` は「先頭 1 個 `x` と残り `xs`」。`:` はリストの「あたまにつける」演算子。

### ガード

「パターンは同じだけど条件で枝分かれ」のときに使う。`if-elif-else` に近い。

```haskell
sign :: Int -> String
sign n
  | n > 0     = "positive"
  | n < 0     = "negative"
  | otherwise = "zero"        -- otherwise は「常に True」のエイリアス
```

> **落とし穴**: パターン／ガードの並びが網羅されていないと、`-Wall` で警告される（デフォルト ON にしてある）。**警告が出たら必ず読む**。実行時エラーの一番の原因。

## 3. `let` と `where` — 一時変数の置き場所

どちらも「式の途中で名前を付ける」ためにある。違いはスコープと書く場所。

### `let ... in ...`（式の中で局所定義）

```haskell
hypot :: Double -> Double -> Double
hypot a b =
  let aa = a * a
      bb = b * b
   in sqrt (aa + bb)
```

`let` で定義した名前は `in` の後でだけ使える。

### `where`（関数定義の末尾に付ける付録）

```haskell
hypot :: Double -> Double -> Double
hypot a b = sqrt (aa + bb)
  where
    aa = a * a
    bb = b * b
```

**結論が先、補助定義が後** に来るので、長めの関数で読みやすい。実務だと `where` をよく使う。

## 4. リスト

`[1, 2, 3]` は内部的に `1 : 2 : 3 : []` と書ける（連結リスト）。

| やりたいこと | 書き方 |
|---|---|
| 先頭に追加 | `0 : [1, 2, 3]` → `[0, 1, 2, 3]` |
| 連結 | `[1, 2] ++ [3, 4]` → `[1, 2, 3, 4]` |
| 最初の要素 | `head [1, 2, 3]` → `1` （※空リストで例外を投げる、後章で扱う） |
| 残り全部 | `tail [1, 2, 3]` → `[2, 3]` |
| 範囲 | `[1..5]` → `[1, 2, 3, 4, 5]` |

### リスト内包表記 — 「集合の内包記法」

数学で「`{ x² | x ∈ N, x が偶数 }`」と書くアレ。

```haskell
[ x * x | x <- [1..10], even x ]
-- → [4, 16, 36, 64, 100]
```

「`[1..10]` から `x` を取り出して、偶数だけに絞り、`x*x` を集める」と左から右に読める。

> **覚えておく価値**: `++` は **左引数の長さに比例** して時間がかかる（O(n)）。リストにバンバン追加するループを書きたくなったら、それは大体 13 章の `Text` か別データ構造の出番。

## 5. 再帰 — Haskell では基本ループが再帰

`for` / `while` がない。代わりに再帰、または高階関数（`map` / `filter` / `foldr`）を使う。

```haskell
sumList :: [Int] -> Int
sumList []     = 0
sumList (x:xs) = x + sumList xs
```

**読み方**: 「空リストの合計は 0。それ以外は『先頭 + 残りの合計』」。再帰の終了条件（base case）と、もう一段呼ぶ条件を分けて書くのが定石。

慣れると `foldr` を使うほうが短いが、まずは自分で再帰を書けることが大事（`Foldable` を理解する土台になる）。

---

## 6. 高階関数 — 「関数を引数に取る関数」

```haskell
map    :: (a -> b) -> [a] -> [b]    -- 各要素を変換
filter :: (a -> Bool) -> [a] -> [a] -- 条件で絞る
foldr  :: (a -> b -> b) -> b -> [a] -> b  -- 畳み込む
```

```haskell
map (* 2) [1, 2, 3]           -- → [2, 4, 6]
filter even [1..10]           -- → [2, 4, 6, 8, 10]
foldr (+) 0 [1, 2, 3, 4]      -- → 10
```

### ラムダ（無名関数）

```haskell
map (\x -> x * x + 1) [1, 2, 3]
--   ^^^^^^^^^^^^^^^^
--   x を取って x*x+1 を返す関数
```

`\` は λ（ラムダ）の代わりの記号。

### セクション — 演算子の部分適用

```haskell
(+ 1)        -- \x -> x + 1   と同じ
(10 -)       -- \x -> 10 - x   と同じ
(* 2)        -- \x -> x * 2   と同じ
```

「丸括弧の中に演算子を入れて、片側だけ書くと、そっちが固定される」。`map (+ 1) [1,2,3]` は `[2,3,4]`。

## 7. `$` と `.` — Haskell 特有の 2 つの演算子

最初は混乱するが、実は超単純。

### `$` は「右側を全部、ひとかたまりで関数の引数にする」

```haskell
print (sqrt (1 + 2 * 3))
print $ sqrt (1 + 2 * 3)        -- 同じ
print $ sqrt $ 1 + 2 * 3        -- これも同じ
```

**括弧を減らすためだけの演算子**。`f $ x = f x` だけど結合が緩いので、右側全体を最後にまとめて適用してくれる。

### `.` は「関数を合成する」

数学の `(f ∘ g)(x) = f(g(x))` のこと。

```haskell
notNull = not . null              -- \xs -> not (null xs) と同じ
```

「`null` してから `not` する関数」を作る。**呼び出さずに合成だけしておく**ところが `$` との違い。

```haskell
-- $ は値を流す
print $ map (* 2) [1, 2, 3]

-- . は関数を組み立てる
let doubleAndShow = show . map (* 2)
print (doubleAndShow [1, 2, 3])
```

> **ポイント**: 関数を **作る** とき `.`、作った関数を **適用する** とき `$`、と覚える。

---

## つまずきやすいポイント

- **インデントは意味がある**: Python と同じく字下げで構造を作る。タブと半角スペースは混ぜない（HLS 推奨設定: スペース 2 つ）
- **演算子の優先順位**: `f x + 1` は `(f x) + 1`、つまり関数適用が一番強い。`f $ x + 1` は `f (x + 1)` ─ ここで `$` が便利
- **型エラーが長い**: 慣れるまで「型が合わない」というだけのエラーが、画面いっぱいに出る。だいたい **シグネチャを先に書く** と問題箇所が絞れる
- **`error` / `undefined` は実行時エラー**: 演習のスタブはこれが入っている。実装すれば消える
